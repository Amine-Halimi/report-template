\section{Définition de l'implémentation}

\subsection{Squelette de l'architecture de l'outil}
Cette section présente le squelette de l’architecture de l’outil proposé. Elle décrit les composants principaux du système et la manière dont ils interagissent afin de permettre l’analyse et la détection de comportements de cryptomining dans des images Docker.

L’objectif de l’outil est de permettre à un utilisateur ou une organisation de fournir une image Docker à l’outil d’analyse. L’outil va effectuer une analyse de cette image et selon le résultat de cette analyse va classifier l’image comme étant légitime, suspecte ou potentiellement malveillante.

\begin{figure}[h!]
    \centering
    \includegraphics[width=0.5\textwidth]{images/architecture_outil.png}
    \caption{Squelette de l'architecture de l'outil}
\end{figure}

\textbf{Interface CLI / API}
Point d’entrée de l’outil. Doit :
\begin{itemize}
    \item Charger l’image Docker
    \item Déclencher l’analyse
    \item Afficher les résultats
\end{itemize}

\textbf{Orchestrateur d’analyse}
Coordonne les différentes phases :
\begin{itemize}
    \item Analyse statique
    \item Analyse dynamique
    \begin{itemize}
        \item Lancement de l’environnement isolé
    \end{itemize}
    \item Accumulation des résultats
\end{itemize}

\textbf{Analyse statique}
\begin{itemize}
    \item Inspection des couches de l’image
    \item Lecture du Dockerfile
    \item Génération d’un SBOM
    \item Extraction des binaires
    \item Analyse des permissions
    \item Recherche de chaînes suspectes (pools, wallets)
\end{itemize}

\textbf{Extraction des IoC}
Détecte :
\begin{itemize}
    \item Présence de XMRig / Minerd
    \item URLs de pools de minage
    \item Commandes suspectes
    \item Scripts shell persistants
    \item Autres indicateurs
\end{itemize}

\textbf{Analyse dynamique}
Exécution du conteneur :
\begin{itemize}
    \item Dans un environnement contrôlé
    \item Avec limites imposées sur le réseau
    \item Monitoring actif
\end{itemize}

\textbf{Monitoring runtime}
Mesure :
\begin{itemize}
    \item Utilisation CPU
    \item Processus actifs
    \item Connexions réseau
    \item Mécanismes de persistance
    \item Tentatives de redémarrage
    \item Autres comportements
\end{itemize}

\textbf{Analyse de corrélation / scoring}
\begin{itemize}
    \item Corrélation entre analyse statique et dynamique
    \item Utilisation d’un score pondéré
    \item Détermination des seuils de classification
\end{itemize}

\textbf{Rapport et classification}
\begin{itemize}
    \item Score final
    \item IoC détectés
    \item Logs d’exécution
    \item Classification
    \item Export dans un format structuré (JSON, CSV, etc.)
\end{itemize}
